Region

  1. Region là gì?

    • Region là khu vực. AWS phân chia các Region độc lập so với các Region khác. ví dụ: US, Euro
    • Mục đích của Region để tăng khả năng chịu lỗi và đạt được sự ổn định tốt nhất.
    • Ví dụ: Trong trường hợp xảy ra lỗi, nếu các instance đều được tạo tạo trong cùng Region thì không có instance để back up

  2. Cách kiểm tra Region?

    • Mở Amazone console tại đây
    • Từ navigation bar, xem các option trong mục Region selector

  3. Cách setup Region default trên AWS cho các instance

    • Mở AWS cli tại Cloud Shell
    • Kiểm tra default region bằng lệnh echo $AWS_DEFAULT_REGION
    • Thay đổi default region bằng lệnh export AWS_DEFAULT_REGION=other-region
